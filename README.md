# CImageOptimizer

CImageOptimizer is a Windows desktop GUI application for optimizing, resizing, and copying image files safely.

## Features

- Select source and output folders from a simple GUI.
- Optimize `.jpg`, `.jpeg`, `.png`, `.webp`, `.bmp`, `.tif`, and `.tiff` files.
- Resize very large images to a maximum width of `1920px`.
- Copy non-image files without modifying them.
- Show live progress and processing logs.
- Build a standalone Windows `.exe`.
- Build a Windows installer with Desktop shortcut, Start Menu shortcut, and uninstall support.

## Project Structure

```text
|   |-- Setup.cs                    # Installer source code
|   `-- Uninstall-CImageOptimizer.ps1

`-- dist/                           # Generated EXE and installer output
```

## Requirements

- Windows
- Python installed and available from PowerShell
- `pip`
- .NET Framework C# compiler, included on most Windows systems

The installer build script uses:

```text
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe
```

If that path is not available, it automatically tries:

```text
C:\Windows\Microsoft.NET\Framework\v4.0.30319\csc.exe
```

Run:


```text

```

## Step 4: Run The App In Development

Run:

```powershell
python main.py
```

Use this command when you want to test the app before creating the `.exe` or installer.

## Step 5: Build The Standalone EXE

Run:

```powershell
pyinstaller --noconsole --onefile main.py
```

After the build finishes, this file is created:

```text
dist\main.exe
```

You can run it directly:

```powershell
.\dist\main.exe
```

## Step 6: Build The Windows Installer

Make sure `dist\main.exe` exists first.

Then run:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\installer\build-installer.ps1
```

After the build finishes, this file is created:

```text
dist\CImageOptimizer-Setup.exe
```

## Step 7: Build Installer From Any Folder

If your PowerShell is currently in another folder, such as `C:\Windows\System32`, use the full path:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File C:\Users\240135357\Downloads\CImageOptimizer\installer\build-installer.ps1
```

The output is still created here:

```text
C:\Users\240135357\Downloads\CImageOptimizer\dist\CImageOptimizer-Setup.exe
```

## Step 8: Install The App

## Troubleshooting

### `installer\build-installer.ps1` does not exist

You are probably not inside the project folder.

Fix:

```powershell
cd C:\Users\240135357\Downloads\CImageOptimizer
```

Then run:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\installer\build-installer.ps1
```

### `pyinstaller` is not recognized

Install the dependencies:

```powershell
pip install -r requirements.txt
```

If it still fails, try:

```powershell
python -m PyInstaller --noconsole --onefile main.py
```

### `dist\main.exe` is missing

Build the standalone app first:

```powershell
pyinstaller --noconsole --onefile main.py
```

### PowerShell execution policy error

Use the provided command with `-ExecutionPolicy Bypass`:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\installer\build-installer.ps1
```

### Installer compiler not found

The installer script requires the .NET Framework C# compiler. Check whether one of these files exists:

```text
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe
C:\Windows\Microsoft.NET\Framework\v4.0.30319\csc.exe
```
